- `adaptPayloadForAnthropic()` does **not** mutate the original payload — it builds a new object. This is safe because `chatHistory` in `chat_ui.js` holds the canonical state.

- Anthropic error branch placed **first** in the response parser (guarded by `isAnthropic`) so it never conflicts with OpenAI error shapes.

## Important: .xsgn signature behaviour

`pi2llm-main.xsgn` has been **deleted** from the repo. It was a PixInsight cryptographic signature (XSGN format) tied to the exact bytes of `pi2llm-main.js`, signed with the upstream developer's private `.xssk` key. Any modification to `pi2llm-main.js` invalidates it; PixInsight then refuses to load the script with "Signature verification failed". The `.xsgn` can only be regenerated by the holder of the matching `.xssk` key using `PixInsight --xssk-file=<key> --sign-xml-file=pi2llm-main.js`. Absence of the file is valid — PixInsight treats unsigned scripts as development-mode and loads them without restriction.

**Do not re-add a `.xsgn` file** unless you have the upstream developer's private key to generate a valid one.
