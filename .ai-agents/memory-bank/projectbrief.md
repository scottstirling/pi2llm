# Project Brief: pi2llm

**Project Name:** pi2llm (LLM Assistant for PixInsight)
**Version:** 2.5.0
**Author:** Scott Stirling, StirlingAstrophoto (scott@stirlingastrophoto.com)
**License:** MIT
**Homepage:** https://scottstirling.github.io/pi2llm/

## Core Requirements & Goals

Integrate Large Language Models natively into the PixInsight astrophotography workspace. The assistant is context-aware: it reads image metadata, FITS headers, astrometric plate-solve data, and live processing history from the active `ImageWindow` and sends it to any OpenAI-compatible LLM API, giving users data-driven processing advice without leaving PixInsight.

## Key Features

- **Context-Aware Analysis:** Extracts FITS headers, astrometry (RA, Dec, resolution, focal length, pixel size), and processing history (both `initialProcessing` and `processing` containers) from the active `ImageWindow`.
- **Visual Analysis (v2+):** Clones the selected view, resizes to LLM-safe dimensions, compresses to JPEG, base64-encodes, and attaches as `image_url` in the multipart chat message. Opt-in globally and per-request.
- **Interactive Chat UI:** PixInsight-native modal `Dialog` with history display, multi-line input, `Ctrl+Enter` shortcut, and export to `.txt` / `.json`.
- **Provider Agnostic:** Supports any OpenAI-compatible endpoint (LM Studio, Ollama, llama.cpp) and Cloudflare AI Gateway. URL, API key, model, temperature, and max tokens are all configurable.
- **Save/Load Configuration Profiles:** Settings saved to `.pi2llm.json` files (API keys stored in plain text — known limitation).
- **PixInsight Update Repository:** `etc/updates.xri` supports distribution via the PixInsight script repository mechanism.
- **Code Signature:** `pi2llm-main.xsgn` is a cryptographic signature generated by PixInsight — never edit manually.
