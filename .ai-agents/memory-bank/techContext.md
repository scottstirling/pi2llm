# Technical Context

## Runtime

- **Engine:** PixInsight JavaScript Runtime (PJSR) — SpiderMonkey embedded in PixInsight. Not Node.js, not a browser.
- **Module system:** C-preprocessor `#include` directives only. No `import`, `export`, or `require()`.
- **Available globals:** `NetworkTransfer`, `ImageWindow`, `View`, `Dialog`, `Settings`, `File`, `ByteArray`, `CoreApplication`, `FileFormat`, `FileFormatInstance`, `MessageBox`, `processEvents()`, and all PJSR standard library objects.
- **Unavailable:** `fetch()`, `XMLHttpRequest`, DOM, `console.log` (use `console.writeln()`), `setTimeout`, `Promise`, `async/await`.

## HTTP / Network

- Only client: `NetworkTransfer` (synchronous, blocks the UI thread).
- **Critical call order:** `setURL()` → `setCustomHTTPHeaders()`. Calling `setURL()` after headers resets headers silently.
- **SSL:** Must call `transfer.setSSL()` explicitly for any `https://` URL.
- **Response decoding:** Always `response.utf8ToString()`, never `.toString()` (breaks on non-ASCII).
- **Outgoing payload:** Always run through `unicodeEscape()` after `JSON.stringify()` before passing to `transfer.post()`.
- **Timeout:** Set to 60 seconds via `transfer.setConnectionTimeout(60)`.

## Settings Persistence

- `Settings.read(key, DataType_*)` / `Settings.write(key, DataType_*, value)` — PixInsight's internal KV store.
- All keys are prefixed `pi2llm/` (constant `SETTINGS_PREFIX`).
- Profile files use extension `.pi2llm.json`; contain API keys in plain text.

## External Library Dependency

- `#include <../src/scripts/AdP/WCSmetadata.jsh>` — provides `ImageMetadata` class for plate-solve data.
- Path is relative to the PixInsight installation directory. On macOS/Windows the resolved path differs from Linux; if `ImageMetadata` is undefined at runtime, this include path is the cause.
- `SETTINGS_MODULE "SOLVER"` must be defined before this include (already set in `image_profile.js`).

## Documentation

- `doc/LLMAssistant.pidoc` — PixInsight native pidoc format source.
- `doc/scripts/` — compiled HTML output generated by PixInsight's pidoc toolchain.
- Help URL: `https://scottstirling.github.io/pi2llm/`

## Testing

- **PJSR tests** (`tests/test_*.js`): Run inside a live PixInsight instance via Script → Run Script File. Each file is self-contained and calls its own entry function.
- **API smoke tests** (`tests/test-*.sh`): Plain `curl` scripts. Require env vars:
  - `OPENAI_API_KEY` for `test-openai.sh`
  - `CF_GATEWAY_AUTH_TOKEN` for `test-cloudflare.sh`
  - No env var needed for `test-ollama.sh` or `test-lmstudio.sh` (local endpoints).
- **No CI, no pre-commit hooks, no linter config.**

## Development Constraints

- UI components: `Dialog`, `Label`, `Edit`, `TextBox`, `PushButton`, `ToolButton`, `CheckBox`, `NumericControl`, `ViewList`, `HorizontalSizer`, `VerticalSizer` — all PixInsight-native.
- No HTML/CSS UI rendering (the `historyBox` uses Qt rich text inside a `TextBox`, not a WebView).
- All dialog execution is modal (`dialog.execute()`); scripts cannot spawn non-blocking windows.
